1.5.0
[Additions]
	- Added (most of) the songs up to 0.6.2 (Weekend 1, Pico Mixes, Erect Songs)
		- Not fully complete yet, but should work fine
		- Also no Blazin yet
	- Added my 'monster-dark'
	- Added StageHelper
		- Contains stage elements and functions of the current stage
		- Can be called from HScripts as 'stage'
			- stage.addElement(name, element); adds a stage element
			- stage.getElement(name); returns stage element of that name
			- stage.removeElement(name, destroy); removes a stage element (destroys on default)
			- stage.addFunction(name, function); adds a function to the stage
			- stage.getFunction(name); gets the function of that name to be later used
			- stage.doFunction(name); uses the function of that name
				- Only works for functions with ZERO input variables (use getFunction instead)
		- Using 'setDefaultZoom();' in a stage hscript will set the stage's default zoom
			- Used for 'ZoomCamera({mode: "stage"});'
		- Will eventually have Z-Index capabilities
	- Added FlxRuntimeShader to PluginManager for HScript use
		- the input in 'new FlxRuntimeShader("adjustColor")', for example, pulls a .frag file of the same name from "assets/shaders/"
		- A path like "assets/images/custom_stages/spooky/sillyShader" also works
		- Currently doesn't have the helper functions, hopefully will be added via HScripts
		- (temporarily, probably) Added DropShadowShader class from modern FNF
			- I'm hoping for a less hard-coded way of implimenting this
			- It has all the helper functions though
	- Added FlxTiledSprite and FlxBackdrop to PluginManager for HScript use
		- Automatically uses FNFAssets to get provided sprite path
	- Added 'onCharacterAdded(char:Character, type:String)' to HScripts
		- Called when a new character is added
		- Currently only works with 'addCharacter' and 'switchCharacter' functions
			- Meaning the pre-generated characters don't automatically call this function
		- The types are 'dad', 'bf', and 'gf'
	- Can now put another character's name for "icons" in 'custom_chars.jsonc' to use their icons instead
	- Added two new variables to the character hscript function 'char.addOffset(name, oppX, oppY, plrX, plrY);'
		- A quality of life made to add offsets to both the player and opponent sides, seperately, in the same function
	- The selected note's array can be viewed in the Note section of ChartingState
	- Added a button to round a note's strum time in the Note section of ChartingState
	- Using arrow keys in AnimationDebug can now modify char offsets
	- Follow cam can now be edited in AnimationDebug
		- 'Force Follow Cam' automatically moves the camera to the character's follow cam
	- Added animationDebugControls to 'assets/data'
		- Will be shown in AnimationDebug eventually
	- Added an option to toggle whether the memory counter shows
	- New Module now outputs any 'picospeaker' charts

[Bug Fixes]

	- Characters using 'loadMappedAnims()' no longer crash when there is no mapped anims

	- 'singLEFT-hold' and 'singRIGHT-hold' now swap when a character is a player
	- Fixed normal icons breaking after switching from an animated icon
	- Freeplay no longer loads song scores before changing difficulty when going to a song that doesn't have the current difficulty selected
	- Charting State now correctly loads songs when a difficulty's default is different from its name
	- Alt Anim and Alt Anim Number now reflect eachother in ChartingState
	- Adding a note to charts in ChartingState no longer adds the deprecated ', false, false' to the end
		- Also doesn't add note length if it's zero, like in modern FNF
	- New Module is more consistent with adding alt animations from new fnf charts

[Additions]
	- Characters can now call updateUV/getUV (universal variables)
	- Added custom icon bop (found at 'assets/images/custom_chars/iconbops/', path may be changed later)
		- Add "iconBop": "$name" to your custom chars
			- Example:
				"bf": {
					"like": "bf",
					"icons": [0,1,2,3],
					"iconbop": "test",
					"colors": ["#149DFF"]
				},
		- or use icon.changeIconBop('$name'); in a hscript
	- Icons now has the variable "healthColors" which contains its icon colors from the custom_chars.jsonc
	- Icon only chars now use "icons" instead of "frames" for consistency
	- New (bool) variable 'currentPlayState.showRatings' can be changed to whether the rating/combo should be shown after hitting a note
	- Added the camera events from the latest FNF
		- FocusCamera({x, y, char, duration, ease});
		- ZoomCamera({zoom, duration, ease, mode});
		- SetCameraBop({rate, intensity});
		- They all work (mostly) like their original counterparts
			- Hopefully, you can understand how to use them
	- Added 'camSpeed' variable to modcharts and such
		- Controls how fast the camera moves (duh)
		- Default value is 0.08 (higher the number, fast the camera moves)
	- Added 'tweenScrollSpeed({speed, duration, ease})' function
	- New function 'skipCountdown()' can be called to cancel the countdown before the song starts
	- Added 'songStart(song)' function
		- Called when the song starts playing
	- Added 'onPause()' and 'onResume()' functions
		- Called when pausing and resuming a song respectively
	- Added 'noteLoaded(note)' hscript function
		- Called when a note is loaded aka about to appear on screen
		- Could be used to modify certain notes depending on its strum time
	- The countdown popups can now be called via 'PlayState.countdownPopup($stage, $image)'
		- $stage (int) is which image it defaults to if the image chosen doesn't exist
			- 0 for ready, 1 for set, 2 for go, anything else shows restart cuz why not
		- $image (assetPath) is what the countdown shows
			- Can be any image you want
			- Leave blank if you just want to use the defaults
	- New 'sing' function for character hscripts
	- Combo is now in camHUD when using 'new judgement layout'
	- Charting State can now be accessed from the pause menu
	- A song's difficulty can be changed from the pause menu
	- Added character browser to Charting State
		- Also works with stages
		- Will probably also do the same for the other things
	- New CoolUtil Function 'CoolUtil.getFilter(filtername, customcolormatrix);'
		- Used to get/make filters via ColorMatrixFilter
			- https://api.openfl.org/openfl/filters/ColorMatrixFilter.html
		- 'Grayscale', 'Invert', 'Deuteranopia', 'Protanopia', 'Tritanopia' are premade from the flixel demo
		- 'Blank' is, well, a blank filter (or normal, default, etc)
		- 'Custom' can be used for your own when provided with a color matrix
	- Added 'pauseTweensOf(obj)' and 'resumeTweensOf(obj)' from base FNF to CoolUtil
	- Note sustains now can have mine/nuke/custom variants
		- For instance, if you're note was 'blue nuke', 'blue nuke hold piece' and 'blue nuke hold end' would be the names for the sustains
		- Added nuke and mine variants for both normal and pixel notes
		- For custom notes, add 'customSustainPath: $path' to the noteinfo for the sustains to change
	- Added a custom sound tray, mainly because the og lagged
		- Also shows "MUTED" in the corner if the game is muted
	- Swap offsets can now be set via 'setSwapOffsets(char:Character, [x, y], additive:Bool);'
		- 'char' determines which character's offsets to change
		- [x, y] is the amount to add to the offset by default (make 'additive' false to set the offsets to that amount)
	- Insts, voices, and modcharts with the suffix "-$difficulty" will be played when on that difficulty
		- Example: 'Inst-erect' will be played when on 'erect' difficulty
	- 'defaults' has been added to custom difficulties
		- This determines which difficulty the song takes it's default assets (characters, stage, ui, etc)
		- Normal difficulty (represented by '') is used unless specified in custom difficulties
		- This also effects the difficulty used in the previous addition ONLY if a default is specified
	- Rank Stars have been improved vastly
		- Now works for all difficulties
		- A star is shown for difficulties not yet completed
		- The star of the current difficulty selected is raised
		- Variants based on difficulty can be added (like records)
			- Uses the defaults from above
			- See erect stars
	- New game icons
	- Game over sounds/music can be modified via a character's hscript
		- char.deathSound is for the death sound (Default: 'fnf_loss_sfx.ogg') (Uses the 'assets/sounds/' folder)
		- char.gameoverMusic is for, well, the game over music (Default: 'gameOver.ogg') (Uses the 'assets/music/' folder)
		- char.gameoverMusicEnd is for when you retry from the game over (Default: 'gameOverEnd.ogg') (Uses the 'assets/music/' folder)
			- The pixel sounds/music are automatically used when char.isPixel = true
